## Current Goal

Build a mobile puzzle game based on the original `infection-blocks.html` prototype.

The target product is an Android-first game with Euclidea-like level progression:

- offline puzzle levels;
- player places initial infection sources;
- pressing start runs infection spread animation;
- final rating depends on how many sources were used;
- early levels free, later levels can be paid unlock;
- no database or self-hosted server for the first prototype.

## Current Level Format

Levels are JSON objects. The editor exports this format.

```json
{
  "id": "world-1-01",
  "packId": "world-1",
  "order": 1,
  "titleKey": "level.world1.01",
  "rows": 10,
  "cols": 7,
  "maxSeeds": 8,
  "free": true,
  "stars": {
    "three": 5,
    "two": 8,
    "one": null
  },
  "holes": [[4, 1]],
  "requiredSeeds": [[0, 0]],
  "blockedSeeds": [[2, 3]]
}
```

Coordinate convention:

- `[row, col]`
- zero-based indexes
- `row` increases downward
- `col` increases rightward

Cell meanings:

- `holes`: empty/nonexistent cells. They do not need to be infected.
- `requiredSeeds`: must be initial infection sources.
- `blockedSeeds`: cannot be initial infection sources, but still must be infected by spread.

Star thresholds:

- `stars.three` is required and means the 3-star source limit.
- `stars.two` can be `null`. If filled, it means the 2-star source limit.
- `stars.one` can be `null`. If filled, it means the 1-star source limit.
- If only `three` is filled, the level is `3 stars / fail`.
- If `three` and `two` are filled, the level is `3 stars / 2 stars / fail`.
- If all three are filled, the level is `3 stars / 2 stars / 1 star / fail`.
- `maxSeeds` is a cached lowest passing threshold, synchronized by the editor.

Current editor behavior:

- Editing the 3-star limit auto-fills:
  - 2-star limit = 3-star limit + 1
  - 1-star limit = 3-star limit + 2
- 2-star and 1-star fields can still be manually edited or cleared.

## Infection Rule

The pure engine is in `src/domain/engine.ts`.

Current rule:

1. Player selects initial infection sources.
2. Required seed cells must be selected.
3. Blocked seed cells cannot be selected.
4. Holes are ignored.
5. On each wave, any non-hole, uninfected cell with at least 2 infected orthogonal neighbors becomes infected.
6. Waves repeat until no new cells are infected.
7. Level succeeds only if all non-hole cells are infected.
8. Stars are calculated from the number of initial sources and the filled star thresholds.

Orthogonal neighbors only:

- up
- down
- left
- right

No diagonal infection.

## Current UI Behavior

There are two modes:

- `輸入關卡` / editor mode
- `試玩` / play mode

Editor mode:

- Switch previous/next existing level.
- Add a new level.
- Edit row/column counts. Minimum is currently 1 row and 1 column.
- Edit level ID.
- Edit star thresholds.
- Toggle free level.
- Select cell tool:
  - normal
  - hole
  - required
  - blocked
- Export/copy/import JSON.

Play mode:

- Required cells appear as already placed infection sources.
- Required cells cannot be removed.
- Blocked cells are red/protected-looking cells.
- Blocked cells cannot receive initial infection sources.
- Blocked cells still need to be infected by spread.
- Clicking ordinary cells only places/removes initial infection sources.
- Infection does not run automatically.
- Press `開始感染` to run wave animation and compute result.
- During infection animation, start/undo/reset/cell editing are disabled.

Small boards:

- If a board is 5x5 or smaller, cells are fixed around 46px instead of expanding to fill the whole screen.

## Persistence

The editor currently stores local working data in browser `localStorage`:

- editor level list
- current level index
- selected locale

This means different browsers or machines will not share edited levels unless the JSON is copied/exported or the source file is updated.

For proper multi-machine work, move finalized level JSON into source-controlled files under `src/data/` later.

## Backend Decision

No database or custom server is needed for the first prototype.

Recommended first release:

- offline bundled levels;
- local progress storage;
- Android first;
- no account system;
- no cloud save;
- no leaderboard;
- no daily challenge.

Add Firebase/Supabase or another hosted service only when these are needed:

- cloud save;
- leaderboard;
- daily levels;
- user-created levels;
- remote config;
- receipt validation beyond basic Play Billing integration.
